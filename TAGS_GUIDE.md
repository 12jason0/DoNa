# 코스 Tags 작성 가이드

온보딩 4단계 선택값 → 코스 `tags` JSON 매핑 관계.
소스: `mobile/src/constants/onboardingData.ts` (웹 `src/constants/onboardingData.ts`와 동일)

---

## 온보딩 4단계 구조

| 단계 | 필드 | 선택 방식 |
|------|------|-----------|
| 1단계 | 분위기 (VIBE) | 1개 선택 |
| 2단계 | 가치관 (VALUE) | 1개 선택 |
| 3단계 | 동행자 (CREW) | 1개 선택 |
| 4단계 | 지역 (REGION) | 복수 선택 |

---

## 1단계 — 분위기 (VIBE) → `concept` + `mood` 태그

| 선택지 | concept 태그 | mood 태그 |
|--------|-------------|----------|
| 따스한 햇살 & 커피 (healing) | 힐링, 맛집탐방, 감성데이트 | 조용한, 감성적 |
| 힙한 네온사인 & 에너지 (hipster) | 이색데이트, 술자리 | 힙한, 활기찬 |
| 은은한 조명 & 와인 (romantic) | 맛집탐방, 야경, 이색데이트 | 프리미엄, 빈티지 |
| 탁 트인 야외 & 산책 (activity) | 공연·전시, 인생샷 | 깔끔한, 이색적인 |

**예시:**
```json
{
  "concept": ["힐링", "맛집탐방", "감성데이트"],
  "mood": ["조용한", "감성적"]
}
```

---

## 2단계 — 가치관 (VALUE) → `concept` + `mood` 태그 (추가)

1단계 결과에 누적 추가됨.

| 선택지 | 추가 concept | 추가 mood |
|--------|-------------|----------|
| 분위기파 (visual) — 분위기·뷰가 중요 | 인생샷 | 감성적 |
| 실속파 (taste) — 맛이 중요 | 맛집탐방 | 빈티지 |

---

## 3단계 — 동행자 (CREW) → `target` 태그

| 선택값 (DB 저장) | id |
|----------------|-----|
| 연인 | partner |
| 친구 | friend |
| 혼자 | solo |
| 소개팅 | blindDate |

**예시:**
```json
{
  "target": ["연인"]
}
```

## 완전한 Tags 예시

온보딩 선택: healing(분위기) + visual(가치관) + 연인(동행자)

```json
{
  "concept": ["힐링", "맛집탐방", "감성데이트", "인생샷"],
  "mood": ["조용한", "감성적"],
  "target": ["연인"]
}
```

---

## 코스에 추가 가능한 Tags (온보딩 외)

관리자가 코스 등록 시 수동으로 추가 가능:

```json
{
  "time": ["오후", "저녁"],
  "theme": ["카페투어", "브런치", "갤러리"],
  "concept": ["야경", "쇼핑"]
}
```

---

## 주의사항

- `concept`, `mood`, `target`: 배열 (복수 값)
- 온보딩 2단계 값은 1단계에 **누적**됨 (덮어쓰기 아님)
- 지역은 `tags`가 아닌 코스의 `location` 필드와 직접 매핑
